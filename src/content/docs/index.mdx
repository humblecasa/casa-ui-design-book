# UI Design Without Guesswork

You’re a senior frontend engineer. You can build anything — but *visual design* (ratios, spacing, sizing, proportions) can turn into endless pixel tweaking.

This guidebook gives you a **system**:

- A small set of **tokens** for spacing, typography, color, and sizing
- Rules that **ban arbitrary values** from shipping
- A **pre-coding workflow** so you decide the UI before writing CSS

> If a value isn’t a token, it doesn’t ship.

## What this solves
- Inconsistent spacing across screens
- Typography hierarchy that “feels off”
- Random component sizes
- UI polish regressions that slip into production
