# Typography tokens

Typography is 90% of UI. Your type choices communicate hierarchy, brand, and professionalism before users read a single word.

## The type scale

Based on a 1.25 ratio (major third) starting from 16px base:

| Token | Size | Line height | Use case |
|-------|------|-------------|----------|
| `--text-xs` | 12px | 1.5 | Captions, helper text, badges |
| `--text-sm` | 14px | 1.5 | Secondary text, metadata, table cells |
| `--text-base` | 16px | 1.5 | Body text, default |
| `--text-lg` | 18px | 1.4 | Lead paragraphs, emphasis |
| `--text-xl` | 20px | 1.4 | Card titles, section headers |
| `--text-2xl` | 24px | 1.3 | Page section titles |
| `--text-3xl` | 30px | 1.2 | Page titles |
| `--text-4xl` | 36px | 1.2 | Hero headlines |
| `--text-5xl` | 48px | 1.1 | Marketing hero, landing pages |

## Font weight tokens

| Token | Weight | Use case |
|-------|--------|----------|
| `--font-normal` | 400 | Body text |
| `--font-medium` | 500 | UI labels, subtle emphasis |
| `--font-semibold` | 600 | Headings, buttons |
| `--font-bold` | 700 | Strong emphasis, hero text |

## Practical combinations

### Page hierarchy
```css
.page-title {
  font-size: var(--text-3xl);
  font-weight: var(--font-semibold);
  line-height: 1.2;
}
.section-title {
  font-size: var(--text-xl);
  font-weight: var(--font-semibold);
  line-height: 1.4;
}
.body {
  font-size: var(--text-base);
  font-weight: var(--font-normal);
  line-height: 1.5;
}
```

### Card patterns
```css
.card-title {
  font-size: var(--text-lg);
  font-weight: var(--font-semibold);
}
.card-description {
  font-size: var(--text-sm);
  color: var(--color-muted);
}
```

### Form elements
```css
.label {
  font-size: var(--text-sm);
  font-weight: var(--font-medium);
}
.input {
  font-size: var(--text-base);
}
.helper-text {
  font-size: var(--text-xs);
  color: var(--color-muted);
}
```

## Line height rules

- **Headings** (24px+): Use 1.1-1.3 (tight)
- **Body text**: Use 1.5-1.6 (comfortable)
- **UI elements**: Use 1.4 (balanced)

Tighter line heights for headings prevent them from looking floaty. Looser line heights for body text improve readability.

## Font stack recommendations

### System fonts (fast, native feel)
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
```

### Modern stacks
```css
/* Clean, professional */
font-family: 'Inter', sans-serif;

/* Distinctive, premium */
font-family: 'Geist', sans-serif;

/* Monospace for code */
font-family: 'JetBrains Mono', 'Fira Code', monospace;
```

## Common mistakes

❌ **Too many sizes**: 8 different font sizes on one page  
✅ **Stick to the scale**: 3-4 sizes max per view

❌ **Weak hierarchy**: Everything looks the same importance  
✅ **Clear contrast**: 2+ steps between hierarchy levels

❌ **All caps everywhere**: MAKES EVERYTHING FEEL LIKE SHOUTING  
✅ **Caps sparingly**: Badges, labels, small UI elements only

❌ **Thin weights for body**: Light text is hard to read  
✅ **400+ for body**: Reserve 300 for large display text only

## Quick hierarchy test

Squint at your screen. Can you still tell what's most important? If everything blurs into sameness, your type hierarchy needs work.
