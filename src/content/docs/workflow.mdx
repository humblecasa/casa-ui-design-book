# Pre-coding workflow (no guesswork)

Design decisions should happen **before** CSS.

## Step 0 — Set constraints (2 min)
- Density: compact vs comfortable
- Max content width: 960 / 1040 / 1200
- Grid: 12-col desktop, 4-col mobile
- Gutters: `32`

## Step 1 — Box layout using only spacing tokens (10–15 min)
Decide:
- Section padding: `48`
- Card padding: `24`
- Major vertical rhythm: `32`
- Inside-card gaps: `12` or `16`

## Step 2 — Typography pass (5–10 min)
Assign roles:
- Page title: `text-2xl` + `lh-tight`
- Section title: `text-xl` + `lh-tight`
- Body: `text-md` + `lh-normal`
- Labels: `text-sm`

## Step 3 — Component pass (5 min)
Pick sizes:
- Default controls: `control-md`
- Table/search: `control-lg` if comfortable

## Step 4 — Color pass (5 min)
- Apply surfaces + text + border
- Apply one accent for primary actions
- Confirm focus/hover/disabled states

## Step 5 — Implement (mechanical)
Translate decisions into code using tokens.

If you feel yourself tweaking pixels, go back a step and update tokens/rules instead.
